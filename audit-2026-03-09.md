## 0. Executive Summary

- 🚫 在 evening-review 修好前，禁止自主产物直接提升为长期记忆

## 12. Source Attribution Policy

所有 diary 条目必须能区分来源，至少加前缀：

| 前缀 | 来源 |
|------|------|
| `[USER-CONVERSATION]` | conversation-logger 自动记录的用户对话 |
| `[MICRO-TRIGGER]` | 微触发思考产物 |
| `[DREAM]` | 梦境思考产物 |
| `[AUTONOMOUS-EXPLORATION]` | 自主探索产物 |
| `[EVENING-REVIEW-SUMMARY]` | 晚间复盘摘要 |
| `[MANUAL]` | memlog.sh 手动记录 |

**原因：**
如果 diary 不区分来源，后续 evening-review 无法安全区分：
- 用户事实
- agent 自主思考
- 探索产物
- 提纯摘要

这会导致系统把 agent 自己生成的内容误提升为长期记忆。

## 13. Memory Promotion Guardrails

**在系统稳定前，自主产物不得直接写入 MEMORY.md。**

### 只允许直接进入长期记忆的内容：
1. 用户明确确认要记住的
2. 长期稳定偏好或约束
3. 被多次真实任务验证过的信息

### 禁止直接进入长期记忆的内容：
1. dream-thinking 输出
2. autonomous-exploration 输出
3. micro-trigger 自由思考
4. 单次闲聊
5. 未经验证的推断

### 临时策略：
- 自主产物可以留在 diary/
- evening-review 可以生成 candidate memory
- candidate memory 在未确认前不得进入 P0 核心记忆
